Maratona de Filmes¶

Ivan Barros¶

Voc quer passar um final de semana assistindo ao mximo de filmes possvel, mas h restries quanto aos horrios disponveis e ao nmero de ttulos que podem ser vistos em cada categoria (comdia, drama, ao, etc).

Entrada: Um inteiro N representando o nmero de filmes disponveis para assistir e N trios de inteiros (H[i], F[i], C[i]), representando a hora de incio, a hora de fim e a categoria do i-simo filme. Alm disso, um inteiro M representando o nmero de categorias e uma lista de M inteiros representando o nmero mximo de filmes que podem ser assistidos em cada categoria.

Sada: Um inteiro representando o nmero mximo de filmes que podem ser assistidos de acordo com as restries de horrios e nmero mximo por categoria.

Heurística Gulosa¶

Um algoritmo guloso qualquer algoritmo que segue a heurstica de resoluo de problemas de fazer a escolha localmente tima em cada estgio. Em muitos problemas, uma estratgia gulosa no produz uma soluo tima, mas uma heurstica gulosa pode produzir solues timas localmente que se aproximam de uma soluo tima global em um perodo de tempo razovel.

Por exemplo, uma estratgia gulosa para o problema do caixeiro viajante (que de alta complexidade computacional) a seguinte heurstica: "A cada passo da jornada, visite a cidade no visitada mais prxima". Esta heurstica no pretende encontrar a melhor soluo, mas termina em um nmero razovel de passos; encontrar uma soluo tima para um problema to complexo normalmente requer muitos passos excessivos. Na otimizao matemtica, os algoritmos gananciosos resolvem problemas combinatrios com propriedades de matrides e fornecem aproximaes de fator constante para problemas de otimizao com a estrutura submodular.

Aleatoriedade¶

Um gerador de nmeros pseudo-aleatrios (RNG) um algoritmo determinstico que gera uma sequncia de nmeros que parece aleatria.

  • determinstico: Um RNG tipicamente recebe como entrada um inteiro seed (que representa uma sequncia de bits "aleatria") e gera uma sequncia de nmeros baseada no seed. Ou seja, o algoritmo determinstico pois gera sempre a mesma sequncia para uma determinada entrada (seed).

Guloso¶

A entrada lida do usurio, onde a primeira linha contm dois inteiros n e m, que correspondem ao nmero total de filmes e o nmero de categorias, respectivamente. A prxima linha contm m inteiros, correspondendo ao nmero mximo de filmes que podem ser escolhidos para cada categoria. As prximas n linhas contm informaes sobre os filmes, cada uma contendo trs inteiros: o tempo de incio, o tempo de trmino e a categoria do filme.

Os filmes so ordenados pelo tempo de trmino e, em seguida, so iterados um por um. Para cada filme, verificado se ainda existem vagas disponveis na categoria correspondente e se o filme compatvel com os filmes j selecionados (ou seja, se o horrio do filme no coincide com o horrio dos filmes selecionados at o momento). Se essas condies forem satisfeitas, o filme adicionado lista de filmes selecionados e o horrio do filme marcado como ocupado. O processo repetido para cada filme at que todas as categorias tenham atingido seu limite de seleo.

void select_movies(vector<Movie>& movies, vector<int>& categories, vector<int>& num_movies, int& marathon_availability) {
    const int num_movies_total = movies.size();

    for (int i = 0; i < num_movies_total; i++) {
        const Movie& movie = movies[i];
        const int category = movie.category;
        const std::bitset<24> schedule = movie.schedule;

        if (num_movies[category - 1] == 0) {
            continue;
        }

        if (marathon_availability == 0 || is_compatible_with_marathon(schedule, marathon_availability)) {
            add_movie_to_marathon(movie, category, num_movies, marathon_availability, categories);
        }
    }
}

bool is_compatible_with_marathon(const std::bitset<24>& schedule, const int marathon_availability) {
    const std::bitset<24> temp_schedule(marathon_availability);
    const std::bitset<24> intersection = temp_schedule & schedule;
    return intersection.none();
}

void add_movie_to_marathon(const Movie& movie, const int category, vector<int>& num_movies, int& marathon_availability, vector<int>& categories) {
    marathon_availability |= movie.schedule.to_ulong();
    categories.push_back(category);
    num_movies[category - 1]--;
}

void select_movies(vector<Movie>& movies, vector<int>& categories, vector<int>& num_movies, int& marathon_availability) {
    const int num_movies_total = movies.size();
    std::default_random_engine rng(10);
    std::uniform_real_distribution<double> dist(0.0, 1.0);

    for (int i = 0; i < num_movies_total; i++) {
        double random_number = dist(rng);
        const Movie& movie = movies[i];
        const int category = movie.category;
        const std::bitset<24> schedule = movie.schedule;

        if (num_movies[category - 1] == 0) {
            continue;
        }

        if (random_number <= 0.25) {
            add_movie_to_marathon(movie, category, num_movies, marathon_availability, categories);
        } else if (marathon_availability == 0 || is_compatible_with_marathon(schedule, marathon_availability)) {
            add_movie_to_marathon(movie, category, num_movies, marathon_availability, categories);
        }
    }
}

Takeaways¶

Ao analisar o desempenho dos programas, possvel destacar que ambos apresentam problemas de vazamento de memria devido falta de desalocao de recursos. No entanto, o programa que utiliza a estratgia aleatria apresenta ainda mais questes relacionadas alocao e liberao de memria, com diversas chamadas repetitivas a funes especficas. preciso otimizar essas chamadas para melhorar a performance desse programa.

Outra questo que precisa ser abordada em ambos os programas a alocao dinmica de memria, que pode ser otimizada.

Ao analisar o desempenho do programa aleatrio, foi identificado que o loop que cria uma lista apenas com filmes disponveis no horrio atual consome muitos recursos, embora seja uma opo melhor do que procurar um filme aleatoriamente. necessrio comparar mais casos para buscar melhorias nesse aspecto.

Por outro lado, no programa guloso, h diversas melhorias possveis, incluindo a alocao dinmica de memria e a reduo no nmero de alocaes e liberaes de recursos.

Vale ressaltar que em todas as anlises realizadas, foi observada uma grande quantidade de chamadas s funes de cin e cout, que realizam diversas requisies de instrues por linha de cdigo. Isso compromete a eficincia do programa, visto que estradas e sadas so extremamente custosas (como vimos em aula)

Comparação de resultados¶

O cdigo abaixo responsvel por testar o desempenho de dois algoritmos diferentes - "greedy" e "randomization" - em diferentes arquivos de entrada com diferentes tamanhos de dados. O objetivo comparar o tempo de execuo e o consumo de memria dos dois algoritmos em diferentes cenrios.

O cdigo cria uma estrutura de dados em forma de dicionrio que armazena os resultados de cada teste. Para cada algoritmo e arquivo de entrada, o cdigo executa o algoritmo dez vezes e mede o tempo de execuo e o consumo de memria para cada execuo. O resultado preliminar de testes indica que o equilbrio entre a abordagem de exploraiton e exploitation fundamental para o sucesso do projeto.
